# ADR-0005: HITL-контур для выплат и чувствительных действий

- **Статус:** Accepted
- **Дата:** 2026-06-18
- **Связанный issue:** [#5](https://github.com/xlabtg/Media_Center/issues/5)

## Контекст

НМЦ использует ИИ для подготовки контента, аналитики, рекомендаций и
автоматизации, но продуктовая и правовая модель требуют, чтобы финальные
чувствительные решения оставались за человеком. Особенно это касается выплат,
статусов участников, массовых публикаций, политик Совета и действий от имени
участника.

## Решение

Принять HITL как обязательный архитектурный контур для чувствительных операций:

- выплаты проходят через HITL Payout Gateway: очередь, окно вето 4-12 часов,
  2FA-подтверждение, исполнение и audit hash;
- Совет и роли с соответствующими правами могут подтвердить, отклонить или
  наложить вето с обязательным обоснованием;
- AI-агенты могут готовить черновики и рекомендации, но не исполняют критичное
  действие без политики, согласия и human approval;
- Policy Manager хранит пороги, окна, лимиты и режимы подтверждения;
- каждое решение в HITL-контуре публикует событие и фиксируется в аудите.

## Последствия

- Сервисы, запускающие чувствительные операции, должны проверять policy перед
  исполнением и отправлять операцию в HITL, если это требуется.
- UI и API должны показывать состояние операции: queued, veto_window_open,
  vetoed, confirmed, executing, executed, failed.
- Автоматическое исполнение допускается только после выполнения правила HITL и
  при отсутствии активного вето.
- Для спорных AI-действий требуется объяснение решения (XAI context), доступное
  Совету в audit trail.
