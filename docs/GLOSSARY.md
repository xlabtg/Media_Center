# Глоссарий терминов НМЦ

Единый словарь терминов проекта. Используется во всей документации, issue, коде и комментариях для исключения разночтений.

---

## Организационные термины

| Термин | Определение |
|--------|-------------|
| **НМЦ** | Народный Медиацентр — программно-организационная платформа распределённого медиа-кооператива. |
| **Пайщик** | Участник кооператива, обладающий паем (долей участия). Бывает ассоциативным и действительным. |
| **Ассоциативный пайщик** | Участник на испытательном этапе: совещательный голос, требование ≥ 1 активности в неделю. |
| **Действительный пайщик** | Полноправный участник: решающий голос, право вето, требование ≥ 10 баллов вклада в квартал. |
| **Пай** | Доля участия пайщика в общей ценности НМЦ; рассчитывается из веса вклада и коэффициента вовлечённости. Не является криптовалютой. |
| **МСЦ** | Метрические Средства Ценности — внутренние единицы измерения ценности вклада («пай»-токен для учёта). **Не криптовалюта**, внутренняя метрика. |
| **Совет** | Высший орган стратегического управления (ранее «Ядро»): стратегия, этика, право вето (кворум 2/3). |
| **Президиум Совета** | Орган надзора и аудита (ранее «Совет»): контроль исполнения, аудит, разрешение споров. |
| **Правление** | Исполнительный орган (Board): оперативное управление, решения простым большинством. |
| **Контур** | Уровень доступа и вовлечённости: Совет (закрытый), Актив (полуоткрытый), Аудитория (открытый). |
| **Тематические дни** | Недельный график тем: Пн — экономика/кооперация, Вт — семья, Ср — технологии/наука, Чт — медиацентр, Пт — образование, Сб — здравоохранение, Вс — идеология/культура. |

## Кибернетические термины

| Термин | Определение |
|--------|-------------|
| **HITL** | Human-in-the-Loop — принцип, при котором ИИ выполняет операции, а человек (Совет) валидирует, задаёт пороги, этику и накладывает вето. |
| **Операционный контур** | Контур обратной связи 1–24 ч: оперативная реакция на события. |
| **Стратегический контур** | Контур обратной связи 24–72 ч: корректировка стратегии. |
| **Адаптивный контур (RL)** | Контур обратной связи 7–30 дней: обучение с подкреплением по KPI (RL-KPI loop). |
| **Окно вето** | Период (4–12 ч, по умолчанию 8 ч), в течение которого Совет может отменить запланированную операцию (например, выплату). |
| **XAI** | Explainable AI — объяснимость решений ИИ для аудита Советом. |

## Технические термины

| Термин | Определение |
|--------|-------------|
| **Tenant (тенант)** | Изолированный экземпляр данных и конфигурации одного медиацентра/сообщества внутри платформы. |
| **`tenant_id`** | Обязательный идентификатор тенанта во всех записях БД, векторах, логах и выплатах. Извлекается из JWT. |
| **Мультитенантность** | Архитектурный принцип: одна платформа обслуживает множество изолированных тенантов; межтенантный доступ запрещён. |
| **Tenant isolation violation** | Нарушение изоляции тенанта; приводит к ответу `403` с кодом `tenant_isolation_violation`. |
| **CGLR** | Content Generator & Link Router — сервис генерации контента и маршрутизации ссылок. |
| **Кв** | Коэффициент (вес) вклада участника: `Кв = min(баллы_участника / средние_баллы_по_Совету; 0.10)`. Ограничен 10 % (`COUNCIL_CAP_KV = 0.10`). |
| **Agentic RAG** | Retrieval-Augmented Generation с агентным поведением (самостоятельный поиск и синтез). |
| **DeepResearch** | Модуль глубокого исследования источников и фактчекинга. |
| **CUA** | Content Agent — ИИ-агент генерации контента. |
| **Voice-to-Chain** | Модуль: голос → локальная транскрипция (Whisper.cpp) → хэш в блокчейн; сырой звук удаляется через 24 ч. |
| **Neuro-Agent Orchestrator** | Оркестратор ИИ-агентов автоматизации (парсинг аудитории, авто-ответы, гигиена контента, аналитика). |
| **Activity Command Center** | Командный центр активности — backend панели Совета/администратора. |
| **L1 / L2 / L3 ссылки** | Уровни ссылок при ротации: L1 — главный CTA (админ), L2 — автор (профиль тенанта), L3 — ротационный партнёр (взвешенно-случайный, вес вклада ≥ 10). |
| **Audit hash** | `SHA256(json.dumps({event_type, tenant_id, points, metadata, timestamp}, sort_keys=True))` — хэш операции для аудита. |
| **Платёжный шлюз РФ** | Интеграция с российскими платёжными системами для выплат пайщикам. |
| **Устойчивость каналов** | Ретраи, очереди, graceful degradation и резервные разрешенные площадки; не включает обход блокировок, ToS или rate limits без отдельного юридического решения. |

## Экономические термины

| Термин | Определение |
|--------|-------------|
| **Распределение 35/17/48** | Разделение дохода: 35 % создателям продукта, 17 % автору идеи, 48 % в пул участников. |
| **Фонды** | Целевые фонды из пула: страховой (10 %), развития (15 %), маркетинга (20 %, приоритет 12 мес.), автоматизации (3 %, приоритет 12 мес.). |
| **Правило 10/10** | 10 % страхового фонда из дохода, затем 10 % фонда развития из остатка — удерживаются до распределения паёв. |
| **`payout_share`** | Доля выплаты участника: `kv_capped / Σ kv_capped`. |
| **Коэффициент вовлечённости** | `текущий_рост_метрик / средний_рост_по_сети`. |
| **Реферальная программа** | Вознаграждение за привлечение: 20 % / 10 % / 5 % для уровней 1 / 2 / 3. |
| **`final_points`** | Итоговые баллы за действие: `round(base × platform_mult × reach_mult × amp_mult, 2)`. |
