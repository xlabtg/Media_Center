# Экономическая модель НМЦ

Документ описывает, как вклад участников превращается в баллы, баллы — в вес вклада (Кв), вес — в долю ценности (пай), и как распределяется доход. Все формулы — основа для реализации **Contribution Ledger & Weight Engine** и **HITL Payout Gateway**.

> ⚠️ **МСЦ (Метрические Средства Ценности) — внутренняя метрика ценности, а не криптовалюта.** Суммы выплат и персональные данные **не записываются** в блокчейн.

---

## 1. Начисление баллов за действия

Итоговые баллы за действие:

```
final_points = round(base × platform_mult × reach_mult × amp_mult, 2)
```

### 1.1. Базовые баллы (`BASE_POINTS`)

| Действие | `base` |
|----------|-------:|
| `idea` (идея) | 0.5 |
| `content_creation` (создание контента) | 12.0 |
| `publish` (публикация) | 5.0 |
| `amplify` (усиление/охват) | 7.0 |
| `engage` (вовлечение) | 3.0 |
| `admin_mod` (администрирование/модерация) | 3.5 |
| `coordinate` (координация) | 2.0 |
| `organize` (организация) | 2.0 |
| `expert_consult` (экспертная консультация) | 3.0 |
| `referral_convert` (реферальная конверсия) | 2.0 |
| `weekly_activity` (недельная активность) | 1.0 |

### 1.2. Множитель площадки (`PLATFORM_MULTIPLIERS`)

| Площадка | Множитель |
|----------|----------:|
| `youtube` | 1.6 |
| `telegram` | 1.5 |
| `gem_space` | 1.4 |
| `dzen` | 1.3 |
| `tenchat` | 1.3 |
| `vk` | 1.2 |
| `ok` | 1.1 |
| `default` | 1.0 |

### 1.3. Множитель охвата (`reach_mult`)

Для `content_creation`:
- охват ≥ 100 000 → ×1.5
- охват ≥ 50 000 → ×1.3
- иначе → ×1.0

### 1.4. Множитель усиления (`amp_mult`)

Для `amplify`:
```
amp_mult += (extra_reach // 50_000) × 0.05
```
(базовое значение `amp_mult = 1.0`, прибавляется по 0.05 за каждые полные 50 000 дополнительного охвата).

---

## 2. Вес вклада (Кв)

Вес вклада участника за период:

```
Кв = min(баллы_участника / средние_баллы_по_Совету ; 0.10)
```

- **Ограничение влияния 10 %** (`COUNCIL_CAP_KV = 0.10`): ни один участник не может иметь вес более 10 %.
- `kv_raw` — значение до ограничения; `kv_capped` — после.

---

## 3. Стоимость пая

```
Стоимость_пая = (Общая_стоимость_НМЦ × (вес_вклада_участника / Σ_вес_всех_действительных))
                × Коэффициент_вовлечённости
```

Коэффициент вовлечённости:

```
Коэффициент_вовлечённости = текущий_рост_метрик / средний_рост_по_сети
```

---

## 4. Доля выплаты

```
payout_share = kv_capped / Σ kv_capped
```

Сумма выплаты участнику = `payout_share × распределяемый_пул`.

---

## 5. Распределение дохода (35 / 17 / 48)

| Доля | Назначение |
|-----:|------------|
| **35 %** | Создателям продукта |
| **17 %** | Автору идеи |
| **48 %** | Пул участников |

### 5.1. Структура пула участников (48 %)

| Доля пула | Фонд / назначение | Приоритет |
|----------:|-------------------|-----------|
| 10 % | Страховой фонд | постоянно |
| 15 % | Фонд развития | постоянно |
| 20 % | Фонд маркетинга | приоритет первые 12 мес. |
| 3 % | Фонд автоматизации | приоритет первые 12 мес. |
| остаток | Распределение в пул / голосование Совета | по решению |

### 5.2. Правило 10/10

Перед распределением паёв удерживаются:
1. **10 % страхового фонда** — из дохода;
2. **10 % фонда развития** — из остатка.

## 6. Жизненный цикл контента (33 % пул)

Распределение пула жизненного цикла контента по стадиям:

| Стадия | Доля | Условия/множители |
|--------|-----:|-------------------|
| Идея | 0.5 % | — |
| Создание | 12 % | ×1.0–1.5 при охвате > 100k |
| Публикация | 5 % | ×1.0–1.3 при 3+ площадках |
| Усиление | 7 % | +0.5 % за каждые 50k охвата |
| Вовлечение | 3 % | — |
| Администрирование | 3.5 % | — |
| Координация | 2 % | — |

## 8. Политика ссылок L1/L2/L3

| Уровень | Назначение | Кто/как |
|---------|------------|---------|
| **L1** | Главный CTA | Назначается администратором |
| **L2** | Ссылка автора | Профиль тенанта/автора |
| **L3** | Ротационный партнёр | Взвешенно-случайный выбор; требование `contribution_weight ≥ 10` |

## 9. Связь с реализацией

- **Contribution Ledger & Weight Engine** реализует разделы 1–4 (см. [modules/contribution-ledger.md](modules/contribution-ledger.md)).
- **HITL Payout Gateway** реализует выплаты по разделам 4–5 с окном вето (см. [modules/hitl-payout-gateway.md](modules/hitl-payout-gateway.md)).
- **CGLR** реализует политику ссылок раздела 8 (см. [modules/cglr.md](modules/cglr.md)).
- **Wallet Module** ведёт учёт МСЦ (см. [modules/wallet.md](modules/wallet.md)).

> Все коэффициенты и доли — конфигурируемые параметры (Policy Manager) и подлежат утверждению Советом. Значения выше — стартовые из исходного регламента.
